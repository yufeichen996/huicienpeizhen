- 后续开发应优先读取本文档，不得默认重新调用 Figma MCP。

- 本文只把已读取结果中能够确认的内容写成确定事实；未保留精确源码值的项目明确标为“未确认”，不得由开发人员自行猜测。
- 当前业务范围仅为上海市范围内医院。

## 2. 当前真实存在的页面与浮层

| 名称 | 类型 | 已确认职责 |
| --- | --- | --- |
| `HomeScreen` | 页面 | 首页欢迎区、搜索视觉入口、Hero、快捷服务、信任指标、推荐陪诊员、合作医院、城市范围提示 |

### 4.2 已确认的视觉规则

- 整体为 Apple/iOS 风格的白蓝医疗视觉。
- 使用 iOS Inset Grouped List 结构和极细分割线。
- 订单筛选使用 iOS Segmented Control。
- Tab Bar 使用半透明白色毛玻璃效果，并保留底部安全区。
- 快捷入口使用蓝、紫、绿、橙四种区分色。
- 主次文本采用黑色、深灰和浅灰层级。

## 6. 四步预约流程

### 6.1 已确认流程

```text
选服务 → 选时间 → 选陪诊员 → 确认 → 预约成功
```

| 步骤 | 已确认内容 | 已确认选择项 | 已确认校验 | 已确认费用逻辑 |
| --- | --- | --- | --- | --- |
| 1. 选服务 | 选择预约服务 | 本次读取结果未保留完整选项 | 必须选择服务后才能继续；具体实现条件未保留 | 服务基础价字段/公式未保留 |
| 2. 选时间 | 选择服务日期和时间 | 具体日期范围与时间段未保留 | 必须选择时间后才能继续；可预约日期规则未保留 | 未确认时间是否影响价格 |
| 3. 选陪诊员 | 选择陪诊员 | 已知至少包含林晓雯、陈建国、王美琳 | 必须选择陪诊员后才能继续；可用性规则未保留 | 已知三人起价分别为 `198`、`158`、`218` 元/次 |
| 4. 确认 | 核对预约与费用并提交 | 具体字段列表未完整保留 | 提交前需要完成前三步；其他校验未保留 | 确认页存在费用明细和违约金提示，但计算公式未保留 |

### 6.2 费用计算边界

当前只能确认：

- 陪诊员存在“起/次”价格。
- 确认步骤展示费用明细。
- 界面存在违约金提示。
- 服务保障文案为“先服务后付款”。

当前不能确认：

- 服务基础价是否与陪诊员起价相加或替换。
- 时长、日期、医院、距离是否产生加价。
- 平台服务费、优惠、预付款、违约金比例或封顶规则。
- 金额取整方式和支付状态机。

在获得明确源代码或业务规则前，开发不得自行实现费用公式。
