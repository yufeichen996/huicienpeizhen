# 汇慈恩陪诊前端原型验收报告

## 版本结论

- 版本号：`v0.1.0-ui-prototype`
- 版本性质：**前端交互原型，不是正式生产版本**。
- 当前可以用于产品演示、内部评审、用户流程验证和后端接口设计。
- 当前不能正式商业上线，原因是微信授权登录、服务端鉴权、正式订单、支付、退款、客服、排班、消息、合规审计、监控和真实数据安全体系尚未接入。

## 完成范围

- 32 个有效页面：4 个 Tab、完整预约、订单、个人中心、搜索、服务、陪诊员、医院及内容设置页面。
- 28 个公共组件（含统一 `app-icon` 和 `app-button`）。
- 完整本地 Mock：6 项服务、3 位陪诊员、8 家医院、患者、优惠券、地址及多状态订单。
- Storage 支持 v1 数据封装、旧裸数据读取兼容、类型异常回退和 Mock 单次初始化。

## Store 清单

| Store | 职责 |
| --- | --- |
| User Store | 游客、模拟登录、个人资料和退出状态 |
| Booking Store | 预约草稿、预选、步骤进度和费用计算 |
| Order Store | 订单快照、状态操作、评价和再次预约 |
| Patient Store | 就诊人、唯一默认项和敏感字段脱敏 |
| Profile Data Store | 收藏、优惠券和常用地址 |

重复的旧 `AppStore.bookingDraft` 已移除。

## Service 清单

- `booking`、`order`、`patient`、`user`
- `service`、`companion`、`hospital`、`search`、`favorite`
- 兼容导出：`booking-service`、`companion-service`、`hospital-service`
- 正式请求预留：`utils/request.ts`，支持 Token、401、超时、requestId 和统一错误；默认 `config/env.ts` 为 `mock`，未配置生产域名或密钥。

## 包体与资源

- 优化前扫描基线：293,424 B。
- 当前文件系统源码估算：371,652 B（约 362.9 KiB）。相较扫描基线增加约 76.4 KiB，主要来自正式 PNG 图标与稳定性基础设施。
- 小程序运行资源共 74,257 B，最大单资源约 2.6 KiB；SVG 图标原稿已移到 `design-assets/icons`，不会进入上传主包。
- 当前规模约为 2 MiB 限值的 17.7%，继续分包不能产生有意义收益，且会增加页面迁移和公共资源重复风险。因此本版本保留单主包并启用 `lazyCodeLoading: requiredComponents`、WXML/WXSS/代码压缩配置。
- 微信最终上传包体必须以开发者工具“代码依赖分析/上传”结果为准。

## 一致性与稳定性结果

- UI 统一为 Apple Blue、`#F2F2F7` 页面背景、26rpx 普通卡片圆角、统一字号和安全区 token。
- 首页主入口、Tab、收藏、消息、设置和订单快捷入口使用统一轻量线性图标，不再渲染 Emoji 功能图标。
- 页面和 Service 不直接调用 `wx.request`；未来正式 API 统一从请求层进入。
- 服务、医院、陪诊员交叉 ID 已自动校验；金额继续以分存储并统一格式化展示。
- 预约提交、订单支付/评价和登录按钮具备防重复处理；导航工具增加短时重复跳转保护。
- 已删除废弃占位页、未使用组件和重复 Booking Store。

## 已知问题与真机待验证

- 尚未运行微信开发者工具 CLI：本机未发现可调用的 CLI 安装路径。
- 必须手工验证 iOS/Android、刘海/灵动岛、小屏、字体放大、Picker、键盘遮挡、毛玻璃降级和真机最终包体。
- 当前已接入自有 SVG 原稿和 72×72 PNG；仍需在真机确认小尺寸清晰度和品牌视觉。
- 医院仅作为陪诊服务地点展示，不代表医院合作、隶属或推荐关系。
- 协议和隐私文本必须由法律与数据合规团队审核。

## 正式能力接入点

- 后端：在 `config/env.ts` 配置环境地址，由 `utils/request.ts` 进入各 Service；补充服务端分页、错误码、重试和可观测性。
- 微信登录：替换 `userService.simulateLogin()`，接入 `wx.login`、服务端 code 换 session、Token 刷新与账号注销。
- 微信支付：在订单待付款状态接入服务端预下单与 `wx.requestPayment`，支付结果以服务端回调为准，不信任前端状态。
- 隐私：补充隐私保护指引、权限用途声明、数据最小化、日志脱敏、留存和删除机制。
