# API 接入状态

更新日期：2026-07-19

状态定义：`已接入` 表示页面或测试已调用真实后端；`后端已备` 表示数据库/仓储层已实现但小程序还未切换；`未接入` 表示仍依赖外部能力或 Mock。

## 合作机构端

| 功能 | 状态 | 说明 |
| --- | --- | --- |
| 密码登录、注销、Session 恢复 | 已接入 | Bearer Session，旧 Demo 请求头默认无效 |
| 业务总览 | 已接入 | 仅本机构聚合 |
| 订单列表和详情 | 已接入 | 服务端强制 institution_id 隔离 |
| 新建、修改、发布订单 | 已接入 | 保存价格快照 |
| 支付确认 | 已接入 | 当前为机构人工确认，不是微信支付 |
| 派单、改派、取消 | 已接入 | 有状态和权限校验 |
| 履约节点、异常、费用查询 | 已接入 | 节点记录按订单和节点绑定 |
| 多操作员细粒度管理 UI | 部分接入 | 数据模型支持多账号；当前页面管理主账号与机构权限 |

## 外部依赖

- 微信公众平台两个小程序的 AppID/Secret；
- 合法请求域名与业务域名备案；
- 微信支付商户号、API v3 密钥、平台证书与回调域名；
- 订阅消息模板 ID；
- 对象存储或受控文件服务；
- 短信/客服渠道；
- 生产监控、告警和异地备份。

在这些依赖完成前，`WECHAT_LOGIN_ENABLED`、`WECHAT_PAYMENT_ENABLED`、`MINIPROGRAM_ENABLED`、`SUBSCRIBE_MESSAGE_ENABLED` 必须保持 `false`。
